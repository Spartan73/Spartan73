PREPROCESSING AND HOW IT WORKS

Preprocessing

Preprocessing is CSS backwards-compatible scripting language(s) that extend the default capabilities of CSS. It enables a developer to use logic in CSS style code, such as variables, nesting, operators, mixins, functions, and even mathematical operations. CSS preprocessors make it easy to automate repetitive tasks, reduce the number of errors and code bloat, create reusable code snippets, and ensure backward compatibility.

An initial example of a CSS Preprocessor was SASS (Syntactically Awesome Style Sheet) and is the oldest that was initially released in 2006. SASS, written in Ruby allows frontend developers to use variables, if/else statements, for/while/each loops, inheritance, and other computational logic in their CSS code.

A newer CSS Preprocessor known as LESS (Leaner Style Sheets) was launched in 2009 and offers many of the features of SASS and is a JavaScript library that extends the default functionalities of CSS and it’s written in JavaScript, so Node.js is needed to install and run the LESS compiler. Holistically, there are two factors of CSS preprocessing: client-side and server-side.

Client-side

Client-side preprocessing, also known as Front-end is what the end-user sees. This type of preprocessing is mainly compiled with HTML, CSS and JavaScript coding. This is the side that end-user experiences on their desktop/laptop PC’s, Tablets and Mobiles phones. Client-side computers (devices) request information from the Server-side computers.

Server-side

Server-side preprocessing, and commonly known as back-end is located on the network side such as server computers, this is the side the end-users do not necessarily see and/or engage with directly. This is the CSS Preprocessing side that is often compiled with Python, JavaScript, PHP, SQL and C# coding, just to name some. Server-side computers serve out information to the Client-side computers.

Mechanisms to LESS

LESS is a dynamic stylesheet language and the indented syntax of LESS is a nested metalanguage that provides a series of mechanisms that allows for real-time compilation via LESS.js and by the browser. The following examples are just some of the LESS mechanisms and will define what they do.

Mixins

Mixins are a group of CSS properties that allow you to use properties of one class for another class and includes class name as its properties. It is a way of mixing in properties from one rule-set into another rule-set in LESS.

Variables

LESS variables are defined with a @ symbol and the values are assigned in the variable with a colon ( : ). Variables are actually "constants" in that they can only be defined once.

Nested Variables

Nesting is used to make code follow some visual hierarchy. It makes the code, simple, clean and less complex. However, it is recommended to be careful while nesting because overly nested rules may cause complexity and it proves hard to maintain.

Operators

Arithmetic operations like addition (+), subtraction ( - ), multiplication ( * ), and/or division ( / ) are supported in LESS. These operations can be done on any number, color and variable as well.

Functions

Functions, such as round function, floor function, ceil function and percentage function, etc. are supported in LESS to manipulate colors and HTML elements aspects in style sheets. LESS maps JavaScript code with manipulation of values and uses predefined functions.
